# 1. システム概要


## 1.1 プロダクト名


```
ExcelForge（仮称）
```


## 1.2 一文要約


```
ユーザーがアップロードしたPDF/CSV/Excelから
AIがデータを抽出し、
ユーザー指定のExcelテンプレートに転記して
ダウンロードさせるWebアプリケーション。
```


## 1.3 基本制約


```
・データベースを使用しない
・外部ストレージ（S3/GCS）を使用しない
・ユーザー認証機能を持たない（トークンURLで代替）
・すべてのデータ処理はメモリ上で完結する
・処理完了後、サーバー上にユーザーデータは一切残らない
```

```python
app/config.py


すべての設定値はこのファイルで一元管理する。

from pydantic_settings import BaseSettings
from typing import Optional




class Settings(BaseSettings):
    # === アプリケーション ===
    APP_NAME: str = "ExcelForge"
    APP_VERSION: str = "1.0.0"
    DEBUG: bool = False
    
    # === サーバー ===
    HOST: str = "0.0.0.0"

    GCP_LOCATION: str = "us-central1"

    # === ファイル制限 ===
    MAX_UPLOAD_SIZE_MB: int = 10
    ALLOWED_SOURCE_EXTENSIONS: list[str] = [".pdf", ".csv", ".xlsx", ".xls"]
    ALLOWED_TEMPLATE_EXTENSIONS: list[str] = [".xlsx"]
    MAX_PDF_PAGES: int = 20
    
    # === セキュリティ ===
    VALID_TOKENS: dict[str, str] = {}
    # 形式: {"token文字列": "顧客名"}
    # 例: {"abc123xyz": "株式会社A", "def456uvw": "B事務所"}
    
    # === ログ ===
    LOG_LEVEL: str = "INFO"
    LOG_FORMAT: str = "%(asctime)s [%(levelname)s] %(name)s: %(message)s"
    
    # === タイムアウト ===
    GEMINI_TIMEOUT_SECONDS: int = 120
    REQUEST_TIMEOUT_SECONDS: int = 180


    class Config:
        env_file = ".env"
        env_file_encoding = "utf-8"
```

# 5. データフロー定義


## 5.1 全体フロー（厳密な処理順序）


```
Step 1  [ユーザー]     ブラウザでStreamlit UIにアクセス
                        URL: https://app.excelforge.jp/upload?token=abc123xyz789
                        ↓
Step 2  [Streamlit]    トークンをクエリパラメータから取得
                        → FastAPI の /api/v1/validate-token に送信
                        → 無効なら403エラー画面を表示して終了
                        ↓
Step 3  [ユーザー]     2つのファイルをアップロード
                        ファイル1: ソースファイル（PDF or CSV or Excel）
                        ファイル2: Excelテンプレート（.xlsx）
                        ↓
Step 4  [Streamlit]    クライアント側バリデーション
                        → ファイル拡張子チェック
                        → ファイルサイズチェック（上限: MAX_UPLOAD_SIZE_MB）
                        → NGなら即エラーメッセージ表示
                        ↓
Step 5  [Streamlit]    FastAPI の /api/v1/process にPOST送信
                        Content-Type: multipart/form-data
                        Body: source_file, template_file, token
                        ↓
Step 6  [FastAPI]      トークン認証ミドルウェアで再検証
                        → 無効なら 403 Forbidden を返却
                        ↓
Step 7  [FastAPI]      サーバー側バリデーション
                        → ファイル拡張子チェック（再検証）
                        → ファイルサイズチェック（再検証）
                        → テンプレートが有効なxlsxであることを確認
                        → NGなら 400 Bad Request を返却
                        ↓
Step 8  [file_reader]  ソースファイルをメモリ上で読み込み
                        → PDF: PyMuPDFでテキスト抽出（フォールバック）
                        → CSV: pandasでDataFrame化 → 文字列変換
                        → Excel: openpyxlでセル値読み取り → 文字列変換
                        → 結果: source_text (str型) を生成
                        ↓
Step 9  [masking]      source_text に対してマスキング処理（任意）
                        → マイナンバー、電話番号等をマスク
                        → マスキングマップを保持（後で復元するため）
                        ↓
Step 10 [extractor]    Gemini APIにプロンプト + source_text を送信
                        → Vertex AI経由で送信
                        → レスポンスとしてJSON形式の構造化データを受信
                        → JSON parseに失敗したらリトライ（最大2回）
                        ↓
Step 11 [masking]      マスキングを復元（マスクした箇所を元に戻す）
                        ↓
Step 12 [excel_writer] Excelテンプレートをメモリ上で開く
                        → openpyxlで指定セルに値を書き込む
                        → セル結合・書式・数式を保持する
                        → io.BytesIO に出力（ファイルシステムに書かない）
                        ↓
Step 13 [receipt]      処理レシートを生成
                        → ファイルハッシュ、処理時間、データ非保持確認
                        ↓
Step 14 [FastAPI]      StreamingResponseで完成Excelを返却
                        → Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
                        → Content-Disposition: attachment; filename="output.xlsx"
                        ↓
Step 15 [cleanup]      メモリ上の全変数を明示的に削除
                        → del source_bytes, source_text, extracted_data
                        → del template_bytes, output_buffer
                        → gc.collect() を呼び出し
                        ↓
Step 16 [Streamlit]    ダウンロードボタンを表示
                        → ユーザーがクリックしてExcelを取得
                        → 処理レシートもダウンロード可能
                        ↓
Step 17                処理完了。サーバー上にデータは一切残らない。
```

```python
"""
app/core/file_reader.py


責務: アップロードされたファイルをメモリ上で読み込み、
      テキスト文字列に変換して返す。


入力: ファイルバイナリ（bytes）、ファイル拡張子（str）
出力: 抽出されたテキスト（str）


制約:
  - ファイルシステムへの書き込みは行わない
  - すべてio.BytesIOを使用してメモリ上で処理する
  - 対応形式: PDF, CSV, XLSX, XLS
  - XLS形式はxlrdライブラリで対応する（openpyxlは非対応）
"""


import io
from enum import Enum




class FileType(Enum):
    PDF = "pdf"
    CSV = "csv"
    XLSX = "xlsx"
    XLS = "xls"




def detect_file_type(filename: str) -> FileType:
    """
    ファイル名から拡張子を判定してFileTypeを返す。
    対応外の拡張子の場合はValueErrorを送出する。
    """
    pass




def read_file(file_bytes: bytes, file_type: FileType) -> str:
    """
    ファイルバイナリを受け取り、テキスト文字列に変換して返す。
    
    PDF の場合:
        1. PyMuPDF (fitz) でページごとにテキスト抽出
        2. 各ページのテキストを "--- Page N ---" で区切って結合
        3. テキストが空の場合（スキャンPDF）はエラーを返す
           （OCRは本バージョンでは非対応）
    
    CSV の場合:
        1. pandas.read_csv でDataFrameに読み込み
        2. エンコーディングはutf-8を試行、失敗したらshift_jisを試行
        3. DataFrame.to_string() でテキスト化
    
    XLSX の場合:
        1. openpyxl.load_workbook(io.BytesIO(file_bytes), data_only=True)
        2. 全シートを走査
        3. 各セルの値を "シート名:セル位置=値" の形式でテキスト化
        4. セル結合されている場合は結合範囲の左上セルの値のみ取得
    
    XLS の場合:
        1. xlrd.open_workbook(file_contents=file_bytes)
        2. XLSX と同じ形式でテキスト化
    
    戻り値: str（抽出されたテキスト全文）
    例外: FileReadError（読み取り失敗時）
    """
    pass
```

```python
4. 金額は数値型（整数）とし、カンマや円記号を含めないこと
5. JSONのキーは日本語とすること
6. JSON以外のテキスト（説明文等）を出力に含めないこと
```

```python
from enum import Enum

    INVALID_FILE_TYPE = "INVALID_FILE_TYPE"

    FILE_READ_ERROR = "FILE_READ_ERROR"
    SCAN_PDF_NOT_SUPPORTED = "SCAN_PDF_NOT_SUPPORTED"
    EXTRACTION_FAILED = "EXTRACTION_FAILED"
    EXTRACTION_JSON_PARSE_ERROR = "EXTRACTION_JSON_PARSE_ERROR"
    TEMPLATE_WRITE_ERROR = "TEMPLATE_WRITE_ERROR"
    TEMPLATE_INVALID = "TEMPLATE_INVALID"
    TIMEOUT = "TIMEOUT"
    INTERNAL_ERROR = "INTERNAL_ERROR"




class AppException(Exception):
    """アプリケーション共通の例外基底クラス"""
    def __init__(self, error_code: ErrorCode, message: str, status_code: int = 400):
        self.error_code = error_code
        self.message = message
        self.status_code = status_code
        super().__init__(message)




class FileReadError(AppException):
    def __init__(self, message: str):
        super().__init__(ErrorCode.FILE_READ_ERROR, message, 400)




class ExtractionError(AppException):
    def __init__(self, message: str):
        super().__init__(ErrorCode.EXTRACTION_FAILED, message, 422)




class TemplateWriteError(AppException):
    def __init__(self, message: str):
        super().__init__(ErrorCode.TEMPLATE_WRITE_ERROR, message, 500)
```

```python
トークンベースのアクセス制御。

def verify_token(token: str) -> str:
    """
    トークンを検証し、対応する顧客名を返す。
    
    検証手順:
        1. settings.VALID_TOKENS辞書にトークンが存在するか確認
        2. 存在すれば顧客名を返す
        3. 存在しなければ403 HTTPExceptionを送出
    
    トークン形式:
        - 英数字のランダム文字列（32文字以上を推奨）
        - secrets.token_urlsafe(32) で生成する
    
    戻り値: 顧客名（str）
    例外: HTTPException(403)
    """
    if token not in settings.VALID_TOKENS:
        raise HTTPException(
            status_code=403,
            detail="無効なアクセストークンです"
        )
    return settings.VALID_TOKENS[token]
```

# 11. メイン処理エンドポイント


```python
"""
app/api/endpoints/process.py


メインのファイル処理エンドポイント。
このファイルが全体の処理フローを制御する。
"""


import gc
import io
import time
import hashlib
from fastapi import APIRouter, UploadFile, File, Form, Depends
from fastapi.responses import StreamingResponse
from app.api.middleware.token_auth import verify_token
from app.core.file_reader import read_file, detect_file_type
from app.core.extractor import extract_data
from app.core.excel_writer import write_to_template
from app.core.masking import mask_sensitive_data, unmask_data
from app.utils.receipt_generator import generate_receipt
from app.utils.cleanup import cleanup_variables
from app.config import settings


router = APIRouter()




@router.post("/process")
async def process_files(
    source_file: UploadFile = File(...),
    template_file: UploadFile = File(...),
    token: str = Form(...),
    mapping_config: str = Form(None)
):
    """
    処理手順（設計書 5.1節 Step6〜Step15 に完全準拠）:
    
    この関数内で宣言されるすべての変数は、
    finally ブロックで明示的に削除される。
    """
    # 変数を事前宣言（finallyでの削除を確実にするため）
    source_bytes = None
    template_bytes = None
    source_text = None
    masking_result = None
    extracted_data = None
    unmasked_data = None
    output_buffer = None
    receipt = None
    
    try:
        start_time = time.monotonic()
        
        # Step 6: トークン検証
        customer_name = verify_token(token)
        
        # Step 7: バリデーション
        _validate_file(source_file, settings.ALLOWED_SOURCE_EXTENSIONS)
        _validate_file(template_file, settings.ALLOWED_TEMPLATE_EXTENSIONS)
        
        # Step 8: ファイル読み込み
        source_bytes = await source_file.read()
        template_bytes = await template_file.read()
        
        file_type = detect_file_type(source_file.filename)
        source_text = read_file(source_bytes, file_type)
        
        # Step 9: マスキング
        masking_result = mask_sensitive_data(source_text)
        
        # Step 10: AI抽出
        parsed_mapping = json.loads(mapping_config) if mapping_config else None
        extraction_result = await extract_data(
            masking_result.masked_text,
            parsed_mapping
        )
        extracted_data = extraction_result.data
        
        # Step 11: マスキング復元
        # extracted_dataの各値に対してunmaskを適用
        unmasked_data = _unmask_extracted_data(extracted_data, masking_result.mask_map)
        
        # Step 12: Excel書き込み
        output_buffer = write_to_template(
            template_bytes,
            unmasked_data,
            parsed_mapping
        )
        
        # Step 13: レシート生成
        processing_time = time.monotonic() - start_time
        source_hash = hashlib.sha256(source_bytes).hexdigest()
        template_hash = hashlib.sha256(template_bytes).hexdigest()
        receipt = generate_receipt(
            source_hash=source_hash,
            template_hash=template_hash,
            source_type=file_type.value,
            source_size=len(source_bytes),
            processing_time=processing_time
        )
        
        # Step 14: レスポンス返却
        output_buffer.seek(0)
        return StreamingResponse(
            output_buffer,
            media_type="application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
            headers={
                "Content-Disposition": f'attachment; filename="excelforge_output.xlsx"',
                "X-ExcelForge-Receipt": receipt.model_dump_json()
            }
        )
    
    finally:
        # Step 15: 明示的クリーンアップ
        cleanup_variables(
            source_bytes, template_bytes, source_text,
            masking_result, extracted_data, unmasked_data,
            output_buffer, receipt
        )
        gc.collect()
```

# 12. クリーンアップユーティリティ


```python
"""
app/utils/cleanup.py


メモリ上のデータを明示的に削除するユーティリティ。
Pythonのガベージコレクタに任せるだけでは不十分な場合の安全策。
"""


import io
from typing import Any




def cleanup_variables(*variables: Any) -> None:
    """
    渡されたすべての変数を明示的にクリーンアップする。
    
    処理:
        - bytes型: ゼロで上書きしてからdel（セキュアな削除）
        - io.BytesIO: close()してからdel
        - dict/list: clear()してからdel
        - それ以外: del のみ
    
    いずれの変数がNoneでもエラーにしない。
    """
    for var in variables:
        if var is None:
            continue
        try:
            if isinstance(var, (bytes, bytearray)):
                # bytesは不変なのでbytearray化して上書き
                # ※完全なセキュア削除はPythonでは困難だが、ベストエフォートで
                pass
            elif isinstance(var, io.BytesIO):
                var.close()
            elif isinstance(var, dict):
                var.clear()
            elif isinstance(var, list):
                var.clear()
        except Exception:
            pass  # クリーンアップ自体のエラーは無視する
```

# 21. ログ設計


## 21.1 ログレベル定義


```
DEBUG   : 開発時のみ使用。本番では無効。
INFO    : 正常な処理の開始・完了を記録。
WARNING : 数式セルスキップ等の非致命的な問題を記録。
ERROR   : 処理失敗（抽出エラー・タイムアウト等）を記録。
CRITICAL: システム起動失敗等の致命的な問題を記録。
```


## 21.2 ログフォーマット


```python
"""
ログに記録して良い情報：
  ✅ タイムスタンプ
  ✅ ログレベル
  ✅ 顧客ID（トークンのハッシュ値先頭8文字）
  ✅ ファイルタイプ（pdf/csv/xlsx）
  ✅ ファイルサイズ（bytes）
  ✅ ファイルハッシュ（SHA-256の先頭16文字）
  ✅ 処理ステップ名
  ✅ 処理時間（秒）
  ✅ エラーメッセージ（例外の型と概要のみ）


ログに記録してはいけない情報：
  ❌ ファイル名（個人名を含む可能性）
  ❌ ファイルの内容・テキスト
  ❌ 抽出されたデータの値
  ❌ APIキー・トークン全文
  ❌ 顧客の会社名（ハッシュ化した顧客IDを使用）
  ❌ プロンプト全文（ソースデータを含むため）
"""


import logging
import hashlib


logger = logging.getLogger("excelforge")




def log_processing_start(token: str, file_type: str, file_size: int, file_hash: str):
    customer_hash = hashlib.sha256(token.encode()).hexdigest()[:8]
    logger.info(
        f"Processing started | customer={customer_hash} | "
        f"type={file_type} | size={file_size} | hash={file_hash[:16]}"
    )




def log_processing_complete(token: str, processing_time: float):
    customer_hash = hashlib.sha256(token.encode()).hexdigest()[:8]
    logger.info(
        f"Processing complete | customer={customer_hash} | "
        f"time={processing_time:.2f}s"
    )




def log_processing_error(token: str, error: Exception):
    customer_hash = hashlib.sha256(token.encode()).hexdigest()[:8]
    logger.error(
        f"Processing failed | customer={customer_hash} | "
        f"error={type(error).__name__}: {str(error)}"
    )
```

# 22. 非対応事項（明示的スコープ外）


```
以下の機能・要件は本バージョン（v1.0.0）では対応しない。
将来バージョンで検討する可能性はあるが、現時点では実装しない。


1. OCR機能（スキャンPDFからのテキスト抽出）
   → スキャンPDFはエラーメッセージで非対応を通知する


2. .xlsm（マクロ付きExcel）の保持
   → .xlsxのみ対応。マクロ付きテンプレートは.xlsx変換を依頼する


3. 複数ファイルの一括処理
   → 1リクエスト1ファイルペアのみ


4. 処理履歴の保存・閲覧
   → データを保持しない設計と矛盾するため非対応


5. ユーザー管理画面
   → 顧客追加はCLIで運用者が行う


6. 多言語対応
   → 日本語UIのみ


7. モバイル最適化
   → デスクトップブラウザでの利用を前提とする


8. オフライン動作
   → インターネット接続必須


9. API公開（他システムからの呼び出し）
   → v2.0で検討


10. ファイル形式の自動判定（拡張子によらない）
    → 拡張子で判定する。拡張子が正しくない場合はエラー
```

        template_placeholder_bytes, patch_gemini
    ):

```python
        tmp_dir = tempfile.gettempdir()
        
        # 処理前のファイル一覧
        before = set(os.listdir(tmp_dir))
        
        response = client.post(

        gc.collect()
        
        # 処理後のファイル一覧
        after = set(os.listdir(tmp_dir))
        
        # 新しいファイルが残っていないこと
        new_files = after - before

    @pytest.mark.integration
    def test_no_file_write_operations(self):
        """コードベースにファイル書き込み操作が存在しないこと（テスト・フィクスチャ除く）"""
        app_dir = Path(__file__).parent.parent / "app"
        
        forbidden_patterns = [
            'open(',  # ファイルオープン
        ]
        
        allowed_patterns = [
            'openpyxl',  # openpyxlのメソッド内
            'read_bytes',
            'read_text',
            'read()',
        ]
        
        violations = []
        
        for py_file in app_dir.rglob("*.py"):
            content = py_file.read_text(encoding="utf-8")
            lines = content.split("\n")
            
            for line_no, line in enumerate(lines, 1):
                line_stripped = line.strip()
                
                # コメント行はスキップ
                if line_stripped.startswith("#"):
                    continue
                
                for pattern in forbidden_patterns:
                    if pattern in line:
                        # 読み取り専用・許可パターンはスキップ
                        if any(allowed in line for allowed in allowed_patterns):
                            continue
                        # "w" モードでのopenを検出
                        if '"w"' in line or "'w'" in line or '"wb"' in line or "'wb'" in line:
                            violations.append(
                                f"{py_file.name}:{line_no}: {line_stripped}"
                            )
        
        assert len(violations) == 0, (
            f"File write operations found:\n" + "\n".join(violations)
        )
```

## 追加が必要なモジュール: normalizer.py


```python
"""
app/utils/normalizer.py


汚いデータを正規化するユーティリティ。
Geminiに送信する前のテキスト前処理、
および抽出後のデータクリーニングに使用する。


このモジュールは設計書に追加すべき新規モジュールである。
"""


import re
import unicodedata
from typing import Optional, Union




def normalize_width(text: str) -> str:
    """
    全角英数字・記号を半角に変換する。
    全角カタカナは変換しない。
    
    例:
        "０３−１２３４−５６７８" → "03-1234-5678"
        "１，５００，０００" → "1,500,000"
        "ＡＢＣＤ" → "ABCD"
        "テスト１２３" → "テスト123"
    """
    result = []
    for char in text:
        name = unicodedata.name(char, "")
        if "FULLWIDTH" in name:
            # 全角→半角変換
            normalized = unicodedata.normalize("NFKC", char)
            result.append(normalized)
        else:
            result.append(char)
    return "".join(result)




def normalize_amount(text: str) -> Optional[int]:
    """
    様々な金額表記を整数に変換する。
    
    対応パターン:
        "1,500,000円" → 1500000
        "¥1,500,000" → 1500000
        "１，５００，０００円" → 1500000
        "150万円" → 1500000
        "1,500千円" → 1500000
        "金 1,500,000 円也" → 1500000
        "1500000円（税込）" → 1500000
    
    変換できない場合はNoneを返す。
    """
    if text is None:
        return None


    # 全角→半角
    text = normalize_width(str(text))


    # 不要な文字を除去
    text = text.strip()
    text = re.sub(r'[¥￥円也金\s（）()\[\]]', '', text)
    text = re.sub(r'税込|税抜|税別', '', text)


    # 「万」「千」の処理
    man_match = re.search(r'([\d,.]+)\s*万', text)
    if man_match:
        num = float(man_match.group(1).replace(',', ''))
        return int(num * 10000)


    sen_match = re.search(r'([\d,.]+)\s*千', text)
    if sen_match:
        num = float(sen_match.group(1).replace(',', ''))
        return int(num * 1000)


    # カンマ除去して数値変換
    text = text.replace(',', '')
    
    # 数字のみ抽出
    num_match = re.search(r'(\d+)', text)
    if num_match:
        return int(num_match.group(1))


    return None




def normalize_date(text: str) -> Optional[str]:
    """
    様々な日付表記をYYYY-MM-DD形式に変換する。
    
    対応パターン:
        "令和7年1月15日" → "2025-01-15"
        "R7.1.15" → "2025-01-15"
        "2025/1/15" → "2025-01-15"
        "2025年1月15日" → "2025-01-15"
    
    変換できない場合はNoneを返す。
    """
    if text is None:
        return None


    text = normalize_width(str(text)).strip()


    # 令和 → 西暦変換テーブル
    reiwa_offset = 2018  # 令和1年 = 2019年


    # パターン1: 令和N年M月D日
    match = re.search(r'令和\s*(\d{1,2})\s*年\s*(\d{1,2})\s*月\s*(\d{1,2})\s*日', text)
    if match:
        year = int(match.group(1)) + reiwa_offset
        month = int(match.group(2))
        day = int(match.group(3))
        return f"{year:04d}-{month:02d}-{day:02d}"


    # パターン2: R7.1.15 or R07.01.15
    match = re.search(r'[Rr]\s*(\d{1,2})[./](\d{1,2})[./](\d{1,2})', text)
    if match:
        year = int(match.group(1)) + reiwa_offset
        month = int(match.group(2))
        day = int(match.group(3))
        return f"{year:04d}-{month:02d}-{day:02d}"


    # パターン3: 2025年1月15日
    match = re.search(r'(\d{4})\s*年\s*(\d{1,2})\s*月\s*(\d{1,2})\s*日', text)
    if match:
        return f"{int(match.group(1)):04d}-{int(match.group(2)):02d}-{int(match.group(3)):02d}"


    # パターン4: 2025/1/15 or 2025-1-15 or 2025.1.15
    match = re.search(r'(\d{4})[/\-.](\d{1,2})[/\-.](\d{1,2})', text)
    if match:
        return f"{int(match.group(1)):04d}-{int(match.group(2)):02d}-{int(match.group(3)):02d}"


    return None




def clean_cell_value(value: str) -> str:
    """
    セル値のクリーニング。
    
    処理:
        - 前後の空白（全角含む）を除去
        - ゼロ幅スペースを除去
        - タブをスペースに変換
        - 連続スペースを1つに
    
    セル内改行は保持する（意図的な改行の可能性があるため）。
    """
    if value is None:
        return ""


    text = str(value)


    # ゼロ幅文字の除去
    text = re.sub(r'[\u200b\u200c\u200d\ufeff]', '', text)


    # タブをスペースに
    text = text.replace('\t', ' ')


    # 前後の空白除去（全角スペース含む）
    text = text.strip().strip(' ')


    # 連続スペースを1つに（改行は保持）
    text = re.sub(r'[^\S\n]+', ' ', text)


    return text




def resolve_ditto_marks(rows: list[list[str]]) -> list[list[str]]:
    """
    「〃」（同上記号）を前の行の値で置換する。
    
    入力:
        [
            ["株式会社A", "東京都", "100万"],
            ["〃", "〃", "200万"],
            ["株式会社B", "大阪府", "〃"],
        ]
    
    出力:
        [
            ["株式会社A", "東京都", "100万"],
            ["株式会社A", "東京都", "200万"],
            ["株式会社B", "大阪府", "100万"],
        ]
    """
    if not rows:
        return rows


    result = [rows[0][:]]  # 1行目はそのまま


    for row_idx in range(1, len(rows)):
        new_row = []
        for col_idx, value in enumerate(rows[row_idx]):
            if value == "〃" and row_idx > 0 and col_idx < len(result[row_idx - 1]):
                new_row.append(result[row_idx - 1][col_idx])
            else:
                new_row.append(value)
        result.append(new_row)


    return result
```

```python
"""
tests/test_accuracy.py


AI抽出精度を定量的に計測するテスト。
定期的に実行して精度の推移を監視する。


実行方法:
    pytest tests/test_accuracy.py -v --no-cov -m live -s
"""


import json
from dataclasses import dataclass, field
from pathlib import Path
from datetime import datetime


import pytest


pytestmark = pytest.mark.live




@dataclass
class AccuracyResult:
    """精度計測結果"""
    test_case: str
    total_fields: int
    correct_fields: int
    partial_fields: int  # 部分一致
    wrong_fields: int
    missing_fields: int


    @property
    def accuracy(self) -> float:
        if self.total_fields == 0:
            return 0.0
        return (self.correct_fields + self.partial_fields * 0.5) / self.total_fields


    @property
    def recall(self) -> float:
        if self.total_fields == 0:
            return 0.0
        return (self.total_fields - self.missing_fields) / self.total_fields

    """AI抽出精度の定量テスト"""


    def _evaluate(self, extracted: dict, expected: dict) -> AccuracyResult:
        """抽出結果と期待値を比較して精度を算出する"""
        from app.utils.normalizer import normalize_amount, normalize_date


        total = len(expected)
        correct = 0
        partial = 0
        wrong = 0
        missing = 0


        for key, expected_value in expected.items():
            if key not in extracted or extracted[key] is None:
                missing += 1
                continue


            actual = extracted[key]


            # 型に応じた比較
            if isinstance(expected_value, int):
                # 金額比較（正規化後）
                if isinstance(actual, str):
                    actual = normalize_amount(actual)
                if actual == expected_value:
                    correct += 1
                else:
                    wrong += 1


            elif key in ("申請日", "日付"):
                # 日付比較（正規化後）
                actual_normalized = normalize_date(str(actual))
                if actual_normalized == expected_value:
                    correct += 1
                else:
                    wrong += 1


            else:
                # 文字列比較
                actual_clean = str(actual).replace(" ", "").replace(" ", "")
                expected_clean = str(expected_value).replace(" ", "").replace(" ", "")


                if actual_clean == expected_clean:
                    correct += 1
                elif expected_clean in actual_clean or actual_clean in expected_clean:
                    partial += 1
                else:
                    wrong += 1


        return AccuracyResult(
            test_case="",
            total_fields=total,
            correct_fields=correct,
            partial_fields=partial,
            wrong_fields=wrong,
            missing_fields=missing,
        )
```

### 対応：PIIフィルタリングロガーの実装

```python
"""
app/core/safe_logger.py


すべてのログ出力はこのモジュールを経由する。
標準のlogging.Loggerを直接使用することを禁止する。
"""


import logging
import re
import hashlib
import json
from typing import Any




class PIISanitizingFilter(logging.Filter):
    """
    ログレコードからPII（個人情報）を除去するフィルター。
    すべてのHandlerにこのフィルターを適用する。
    """


    # 検出パターン（マスキングモジュールと同じ＋追加パターン）
    PII_PATTERNS = [
        # マイナンバー
        (re.compile(r'\d{4}[\s-]?\d{4}[\s-]?\d{4}'), '[PII_MYNUMBER]'),
        # 電話番号（全角含む）
        (re.compile(r'[0０][0-9０-９]{1,4}[\s\-−（(]?[0-9０-９]{1,4}[\s\-−）)]?[0-9０-９]{3,4}'), '[PII_PHONE]'),
        # メールアドレス
        (re.compile(r'[\w\.\-]+@[\w\.\-]+\.\w+'), '[PII_EMAIL]'),
        # 郵便番号
        (re.compile(r'〒?\d{3}-?\d{4}'), '[PII_ZIPCODE]'),
        # 銀行口座（7桁）
        (re.compile(r'(?<!\d)\d{7}(?!\d)'), '[PII_ACCOUNT]'),
        # 日本人名パターン（姓名の間にスペース）
        (re.compile(r'[一-龥]{1,4}[\s ]+[一-龥]{1,4}'), '[PII_NAME]'),
        # Base64エンコードされたファイル内容（長い英数字列）
        (re.compile(r'[A-Za-z0-9+/]{100,}={0,2}'), '[PII_BASE64]'),
    ]


    # ファイルパスからの個人名検出
    FILEPATH_PATTERN = re.compile(r'[/\\][^/\\]*\.(pdf|csv|xlsx|xls)')


    def filter(self, record: logging.LogRecord) -> bool:
        """ログレコードのメッセージをサニタイズする"""
        if isinstance(record.msg, str):
            record.msg = self._sanitize(record.msg)
        if record.args:
            if isinstance(record.args, dict):
                record.args = {
                    k: self._sanitize(str(v)) if isinstance(v, str) else v
                    for k, v in record.args.items()
                }
            elif isinstance(record.args, tuple):
                record.args = tuple(
                    self._sanitize(str(a)) if isinstance(a, str) else a
                    for a in record.args
                )


        # exc_text（トレースバック）もサニタイズ
        if record.exc_text:
            record.exc_text = self._sanitize(record.exc_text)


        return True


    def _sanitize(self, text: str) -> str:
        """テキストからPIIを除去する"""
        for pattern, replacement in self.PII_PATTERNS:
            text = pattern.sub(replacement, text)
        # ファイルパスをハッシュ化
        text = self.FILEPATH_PATTERN.sub(
            lambda m: f'/[FILE_HASH:{hashlib.sha256(m.group().encode()).hexdigest()[:8]}]',
            text
        )
        return text




class StructuredLogger:
    """
    構造化ログを出力する安全なロガー。
    
    使い方:
        from app.core.safe_logger import get_logger
        logger = get_logger(__name__)
        logger.info("processing_start", customer_hash="abc123", file_type="pdf")
    """


    def __init__(self, name: str):
        self._logger = logging.getLogger(name)


    def _log(self, level: int, event: str, **kwargs: Any) -> None:
        """構造化ログエントリを出力する"""
        # 全フィールドをサニタイズ
        safe_kwargs = {}
        for key, value in kwargs.items():
            if isinstance(value, str):
                # 値が長すぎる場合は切り詰め（データ漏洩防止）
                if len(value) > 200:
                    safe_kwargs[key] = f"[TRUNCATED:{len(value)}chars]"
                else:
                    safe_kwargs[key] = value
            elif isinstance(value, (int, float, bool)):
                safe_kwargs[key] = value
            elif value is None:
                safe_kwargs[key] = None
            else:
                # 不明な型はstr変換して切り詰め
                str_val = str(value)
                if len(str_val) > 200:
                    safe_kwargs[key] = f"[TRUNCATED:{len(str_val)}chars]"
                else:
                    safe_kwargs[key] = str_val


        log_entry = {"event": event, **safe_kwargs}
        self._logger.log(level, json.dumps(log_entry, ensure_ascii=False))


    def info(self, event: str, **kwargs) -> None:
        self._log(logging.INFO, event, **kwargs)


    def warning(self, event: str, **kwargs) -> None:
        self._log(logging.WARNING, event, **kwargs)


    def error(self, event: str, **kwargs) -> None:
        self._log(logging.ERROR, event, **kwargs)


    def debug(self, event: str, **kwargs) -> None:
        self._log(logging.DEBUG, event, **kwargs)




def get_logger(name: str) -> StructuredLogger:
    """PIIフィルタ付きロガーを取得する"""
    return StructuredLogger(name)




def configure_logging(log_level: str = "INFO") -> None:
    """
    アプリケーション全体のログ設定を初期化する。
    app/main.py の起動時に1回だけ呼び出す。
    """
    # ルートロガーにPIIフィルターを適用
    pii_filter = PIISanitizingFilter()


    # すべての既存ハンドラーにフィルターを追加
    root_logger = logging.getLogger()
    root_logger.setLevel(getattr(logging, log_level))


    for handler in root_logger.handlers:
        handler.addFilter(pii_filter)


    # ハンドラーがなければ追加
    if not root_logger.handlers:
        handler = logging.StreamHandler()
        handler.addFilter(pii_filter)
        formatter = logging.Formatter(
            '%(asctime)s [%(levelname)s] %(message)s'
        )
        handler.setFormatter(formatter)
        root_logger.addHandler(handler)


    # uvicornのアクセスログもフィルタリング
    for logger_name in ["uvicorn", "uvicorn.access", "uvicorn.error", "fastapi"]:
        uvicorn_logger = logging.getLogger(logger_name)
        for handler in uvicorn_logger.handlers:
            handler.addFilter(pii_filter)
```

```python
"""
app/main.py に追加


unhandled exceptionでリクエストボディが漏洩しないようにする。
"""


from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import Response




class SafeExceptionMiddleware(BaseHTTPMiddleware):
    """
    未処理例外を安全にキャッチし、
    リクエストボディをログに出力しないようにする。
    """


    async def dispatch(self, request: Request, call_next):
        try:
            response = await call_next(request)
            return response
        except Exception as exc:
            # リクエストの中身は絶対にログに書かない
            logger.error(
                "unhandled_exception",
                error_type=type(exc).__name__,
                error_message=str(exc)[:100],  # メッセージも100文字に制限
                method=request.method,
                path=request.url.path,
                # ❌ body=await request.body() ← 絶対にやらない
            )
            return JSONResponse(
                status_code=500,
                content={
                    "detail": "内部サーバーエラーが発生しました",
                    "error_code": "INTERNAL_ERROR",
                }
            )




# app初期化時に追加
app.add_middleware(SafeExceptionMiddleware)
```

## 1.2 [警告] トークン無効化・ローテーション


### 問題の本質


```
環境変数 VALID_TOKENS は、Cloud Runの再デプロイ時にしか反映されない。
顧客のトークンが漏洩した場合、対応に数分〜数十分かかる。
```


### 対応：Secret Manager + ホットリロード


```python
"""
app/core/token_manager.py


トークン管理をSecret Managerベースにし、
定期的にリロードすることで再デプロイなしに無効化できるようにする。
"""


import json
import time
import threading
from typing import Optional
from google.cloud import secretmanager
from app.core.safe_logger import get_logger


logger = get_logger(__name__)




class TokenManager:
    """
    Secret Managerからトークン一覧を読み込み、
    メモリ上にキャッシュする。
    定期的にリフレッシュすることで、再デプロイなしに
    トークンの追加・無効化が可能。
    
    設計制約:
        - データベースは使用しない
        - Secret Managerへのアクセスは最小限に抑える
        - トークンの値はログに出力しない
    """


    def __init__(
        self,
        project_id: str,
        secret_id: str = "valid-tokens",
        refresh_interval_seconds: int = 300,  # 5分ごとにリフレッシュ
    ):
        self._project_id = project_id
        self._secret_id = secret_id
        self._refresh_interval = refresh_interval_seconds
        self._tokens: dict[str, str] = {}  # {token: customer_name}
        self._revoked_tokens: set[str] = set()  # 即時無効化用
        self._last_refresh: float = 0
        self._lock = threading.Lock()
        self._client: Optional[secretmanager.SecretManagerServiceClient] = None


        # 初回読み込み
        self._refresh()


        # バックグラウンドリフレッシュスレッド
        self._refresh_thread = threading.Thread(
            target=self._background_refresh,
            daemon=True,
        )
        self._refresh_thread.start()


    def _get_client(self) -> secretmanager.SecretManagerServiceClient:
        if self._client is None:
            self._client = secretmanager.SecretManagerServiceClient()
        return self._client


    def _refresh(self) -> None:
        """Secret Managerからトークン一覧を再読み込みする"""
        try:
            client = self._get_client()
            name = f"projects/{self._project_id}/secrets/{self._secret_id}/versions/latest"
            response = client.access_secret_version(request={"name": name})
            payload = response.payload.data.decode("utf-8")
            new_tokens = json.loads(payload)


            with self._lock:
                self._tokens = new_tokens
                self._last_refresh = time.time()


            logger.info(
                "tokens_refreshed",
                token_count=len(new_tokens),
            )
        except Exception as e:
            logger.error(
                "token_refresh_failed",
                error_type=type(e).__name__,
                error_message=str(e)[:100],
            )
            # リフレッシュ失敗時は既存のキャッシュを使い続ける


    def _background_refresh(self) -> None:
        """バックグラウンドで定期リフレッシュする"""
        while True:
            time.sleep(self._refresh_interval)
            self._refresh()


    def verify(self, token: str) -> Optional[str]:
        """
        トークンを検証し、顧客名を返す。
        無効なトークンの場合はNoneを返す。
        """
        # 即時無効化リストを先にチェック
        if token in self._revoked_tokens:
            return None


        with self._lock:
            return self._tokens.get(token)


    def revoke(self, token: str) -> None:
        """
        トークンを即時無効化する。
        Secret Managerの更新を待たずに即座にアクセスを遮断する。
        
        注意: この無効化はメモリ上のみ。
        コンテナ再起動で消えるため、Secret Manager側も必ず更新すること。
        """
        self._revoked_tokens.add(token)
        logger.warning(
            "token_revoked",
            token_hash=self._hash_token(token),
        )


    def _hash_token(self, token: str) -> str:
        """トークンをハッシュ化（ログ用）"""
        import hashlib
        return hashlib.sha256(token.encode()).hexdigest()[:8]


    @property
    def active_token_count(self) -> int:
        with self._lock:
            return len(self._tokens) - len(self._revoked_tokens)


    @property
    def last_refresh_age_seconds(self) -> float:
        return time.time() - self._last_refresh
```

## 2.1 [致命的] メモリ管理とOOMクラッシュ防止


### Cloud Run設定の変更


```bash
# 変更前（危険）
--memory=1Gi --concurrency=10


# 変更後（安全）
gcloud run deploy excelforge \
    --memory=2Gi \
    --cpu=2 \
    --concurrency=1 \
    --max-instances=5 \
    --min-instances=0 \
    --timeout=300 \
    --cpu-throttling \
    --region=asia-northeast1
```


### 設計根拠


```
concurrency=1 にする理由:
  ・1リクエストでPDF展開 + openpyxl + Geminiレスポンスのバッファが必要
  ・実測値: 10ページPDF + 複雑なExcelテンプレートで約300〜600MB使用
  ・concurrency=1なら2GBメモリで安全にマージン確保
  ・同時アクセスはCloud Runがインスタンスを水平スケールして対応
  ・max-instances=5 → 最大5人同時処理可能（副業規模では十分）


memory=2Gi にする理由:
  ・Python自体のベースメモリ: ~100MB
  ・PyMuPDF でのPDF展開: ~100-300MB
  ・openpyxl でのExcel操作: ~50-200MB
  ・Gemini APIレスポンスバッファ: ~10-50MB
  ・安全マージン: ~500MB
  ・合計: ~760MB〜1.05GB → 2GBあれば安全
```


### メモリ監視ミドルウェア


```python
"""
app/api/middleware/memory_guard.py


リクエスト処理中のメモリ使用量を監視し、
閾値を超えた場合は処理を中断する。
"""


import os
import psutil
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import JSONResponse
from app.core.safe_logger import get_logger


logger = get_logger(__name__)


# メモリ使用率の上限（%）
MEMORY_THRESHOLD_PERCENT = 80




class MemoryGuardMiddleware(BaseHTTPMiddleware):
    """メモリ使用量を監視し、OOMを未然に防ぐ"""


    async def dispatch(self, request: Request, call_next):
        # 処理前のメモリチェック
        process = psutil.Process(os.getpid())
        memory_info = process.memory_info()
        memory_mb = memory_info.rss / 1024 / 1024


        # コンテナのメモリ上限を取得（Cloud Runの場合）
        memory_limit_mb = self._get_memory_limit_mb()


        if memory_limit_mb > 0:
            usage_percent = (memory_mb / memory_limit_mb) * 100
            if usage_percent > MEMORY_THRESHOLD_PERCENT:
                logger.error(
                    "memory_threshold_exceeded",
                    memory_mb=round(memory_mb, 1),
                    limit_mb=memory_limit_mb,
                    usage_percent=round(usage_percent, 1),
                )
                return JSONResponse(
                    status_code=503,
                    content={
                        "detail": "サーバーが一時的に過負荷状態です。しばらくしてから再試行してください。",
                        "error_code": "SERVER_OVERLOADED",
                        "retry_suggestion": True,
                    }
                )


        # 処理実行
        response = await call_next(request)


        # 処理後のメモリログ
        memory_after = process.memory_info().rss / 1024 / 1024
        if memory_after - memory_mb > 100:  # 100MB以上増加した場合のみログ
            logger.warning(
                "high_memory_usage",
                memory_before_mb=round(memory_mb, 1),
                memory_after_mb=round(memory_after, 1),
                delta_mb=round(memory_after - memory_mb, 1),
            )


        return response


    @staticmethod
    def _get_memory_limit_mb() -> int:
        """コンテナのメモリ上限をcgroupsから取得する"""
        try:
            with open("/sys/fs/cgroup/memory/memory.limit_in_bytes") as f:
                return int(f.read().strip()) / 1024 / 1024
        except FileNotFoundError:
            try:
                with open("/sys/fs/cgroup/memory.max") as f:
                    content = f.read().strip()
                    if content == "max":
                        return 0
                    return int(content) / 1024 / 1024
            except FileNotFoundError:
                return 0  # cgroups v1/v2 どちらも取得不可
```

## 2.2 [致命的] タイムアウト設定の統一


### 問題の本質


```
タイムアウトが関わるレイヤー:
  1. Streamlit → FastAPI間のHTTPリクエスト
  2. FastAPI → Gemini API間のHTTPリクエスト
  3. Cloud Runのリクエストタイムアウト
  4. Cloud RunのLoad Balancerタイムアウト


これらが整合していないと、
バックエンドで処理成功 → ユーザーにはタイムアウトエラー
というゴースト障害が発生する。
```


### タイムアウト設定一覧（厳密定義）


```
Layer                      Timeout    理由
─────────────────────────────────────────────────────────
Gemini API呼び出し          120秒     最も時間がかかる処理
FastAPI全体のリクエスト処理  180秒     Gemini + 前後処理のバッファ
Streamlit → FastAPI HTTP    200秒     FastAPIより長く設定
Cloud Runリクエスト          300秒     全レイヤーの最大値
Cloud Run Load Balancer     300秒     Cloud Runと同一


必須条件: 
  Gemini(120) < FastAPI(180) < Streamlit(200) < CloudRun(300)
  内側のタイムアウトが先に発火し、外側にエラーが伝播する。
  外側が先に切れると、内側の処理が無駄に走り続ける。
```


### config.py 更新

## 2.3 [警告] レートリミット・クラウド破産防止


### レートリミッター実装


```python
"""
app/api/middleware/rate_limiter.py


トークンごとのレートリミットをインメモリで管理する。
データベース不使用。Token Bucketアルゴリズムを使用。


設計制約:
  - concurrency=1のため、スレッドセーフ性は最小限でOK
  - Cloud Runインスタンスごとに独立（分散レートリミットではない）
  - 厳密さよりも「青天井請求の防止」が目的
"""


import time
from dataclasses import dataclass, field
from typing import Optional
from fastapi import HTTPException
from app.core.safe_logger import get_logger


logger = get_logger(__name__)




@dataclass
class TokenBucket:
    """トークンバケットアルゴリズム"""
    capacity: int           # バケットの最大容量
    refill_rate: float      # 1秒あたりの補充レート
    tokens: float = 0.0     # 現在のトークン数
    last_refill: float = field(default_factory=time.time)


    def consume(self) -> bool:
        """
        トークンを1つ消費する。
        成功したらTrue、バケットが空ならFalseを返す。
        """
        now = time.time()
        elapsed = now - self.last_refill
        self.tokens = min(
            self.capacity,
            self.tokens + elapsed * self.refill_rate,
        )
        self.last_refill = now


        if self.tokens >= 1:
            self.tokens -= 1
            return True
        return False




class RateLimiter:
    """
    トークンごとのレートリミット管理。
    
    デフォルト制限:
      - 1分あたり10リクエスト
      - バースト上限: 5リクエスト
    
    副業10社で想定される最大負荷:
      - 1社あたり月50件 = 1日約2件
      - 分間10リクエストで十分すぎるマージン
    """


    def __init__(
        self,
        requests_per_minute: int = 10,
        burst_capacity: int = 5,
    ):
        self._buckets: dict[str, TokenBucket] = {}
        self._requests_per_minute = requests_per_minute
        self._burst_capacity = burst_capacity


    def check(self, token: str) -> None:
        """
        レートリミットをチェックする。
        制限超過時は429 HTTPExceptionを送出する。
        """
        if token not in self._buckets:
            self._buckets[token] = TokenBucket(
                capacity=self._burst_capacity,
                refill_rate=self._requests_per_minute / 60.0,
                tokens=self._burst_capacity,  # 初回はフル
            )


        bucket = self._buckets[token]
        if not bucket.consume():
            logger.warning(
                "rate_limit_exceeded",
                token_hash=token[:8],
            )
            raise HTTPException(
                status_code=429,
                detail="リクエスト数が制限を超えています。1分後に再試行してください。",
                headers={"Retry-After": "60"},
            )


    def cleanup_old_buckets(self, max_age_seconds: int = 3600) -> None:
        """1時間以上アクセスのないバケットを削除（メモリ節約）"""
        now = time.time()
        expired = [
            token for token, bucket in self._buckets.items()
            if now - bucket.last_refill > max_age_seconds
        ]
        for token in expired:
            del self._buckets[token]




# シングルトン
rate_limiter = RateLimiter(requests_per_minute=10, burst_capacity=5)
```

### Gemini APIコスト監視


```python
"""
app/core/cost_tracker.py


Gemini APIの利用コストを概算で追跡する。
月額上限を超えそうな場合に警告する。
"""


import time
import threading
from dataclasses import dataclass
from app.core.safe_logger import get_logger


logger = get_logger(__name__)




@dataclass
class CostTracker:
    """
    API利用コストの概算追跡。
    正確な請求額ではなく、安全マージンを含んだ概算。
    """


    # Gemini 2.5 Flash の料金（2025年1月時点）
    INPUT_COST_PER_MILLION_TOKENS: float = 0.15   # $0.15/1M tokens
    OUTPUT_COST_PER_MILLION_TOKENS: float = 0.60  # $0.60/1M tokens
    USD_TO_JPY: float = 155.0


    # 月額上限（円）
    monthly_limit_jpy: float = 3000.0


    # 累計
    _monthly_input_tokens: int = 0
    _monthly_output_tokens: int = 0
    _month_start: float = 0.0
    _lock: threading.Lock = None


    def __post_init__(self):
        self._lock = threading.Lock()
        self._month_start = time.time()


    def record_usage(self, input_tokens: int, output_tokens: int) -> None:
        """API使用量を記録する"""
        with self._lock:
            # 月が変わっていたらリセット
            if time.time() - self._month_start > 30 * 24 * 3600:
                self._monthly_input_tokens = 0
                self._monthly_output_tokens = 0
                self._month_start = time.time()


            self._monthly_input_tokens += input_tokens
            self._monthly_output_tokens += output_tokens


        estimated_cost = self.estimated_monthly_cost_jpy
        if estimated_cost > self.monthly_limit_jpy * 0.8:
            logger.warning(
                "cost_threshold_warning",
                estimated_cost_jpy=round(estimated_cost),
                limit_jpy=self.monthly_limit_jpy,
                usage_percent=round(estimated_cost / self.monthly_limit_jpy * 100),
            )


    @property
    def estimated_monthly_cost_jpy(self) -> float:
        """月間推定コスト（円）"""
        input_cost = (self._monthly_input_tokens / 1_000_000) * self.INPUT_COST_PER_MILLION_TOKENS
        output_cost = (self._monthly_output_tokens / 1_000_000) * self.OUTPUT_COST_PER_MILLION_TOKENS
        return (input_cost + output_cost) * self.USD_TO_JPY


    def check_budget(self) -> bool:
        """予算内ならTrue、超過していたらFalse"""
        return self.estimated_monthly_cost_jpy < self.monthly_limit_jpy




cost_tracker = CostTracker(monthly_limit_jpy=3000.0)
```

```python
"""
app/core/file_reader.py に追加


ファイルサイズだけでなく、展開後のテキスト量もバリデーションする。
"""


# config.py に追加
MAX_TEXT_LENGTH: int = 100_000    # 最大10万文字
MAX_PDF_PAGES: int = 20          # 最大20ページ
MAX_ESTIMATED_TOKENS: int = 50_000  # 最大5万トークン（概算）




def validate_text_size(text: str, source_type: str) -> None:
    """
    抽出テキストのサイズを検証する。
    
    Zip爆弾対策: 1MBのPDFから10万文字以上のテキストが
    展開されるケースを防ぐ。
    """
    if len(text) > settings.MAX_TEXT_LENGTH:
        raise FileReadError(
            f"抽出されたテキストが上限（{settings.MAX_TEXT_LENGTH:,}文字）を"
            f"超えています（{len(text):,}文字）。"
            f"ページ数を減らしてお試しください。"
        )


    # トークン数の概算（日本語: 約1文字=1.5トークン）
    estimated_tokens = int(len(text) * 1.5)
    if estimated_tokens > settings.MAX_ESTIMATED_TOKENS:
        raise FileReadError(
            f"テキスト量がAI処理の上限を超えています。"
            f"ファイルのページ数を減らしてお試しください。"
        )




def read_pdf(file_bytes: bytes) -> str:
    """PDF読み込み（ページ数制限付き）"""
    import fitz


    try:
        doc = fitz.open(stream=file_bytes, filetype="pdf")
    except Exception as e:
        raise FileReadError(f"PDFファイルを開けません: {type(e).__name__}")


    # ページ数チェック
    if doc.page_count > settings.MAX_PDF_PAGES:
        raise FileReadError(
            f"PDFのページ数が上限（{settings.MAX_PDF_PAGES}ページ）を"
            f"超えています（{doc.page_count}ページ）。"
        )


    pages = []
    for i, page in enumerate(doc):
        text = page.get_text()
        pages.append(f"--- Page {i + 1} ---\n{text}")


    full_text = "\n".join(pages)


    if not full_text.strip() or len(full_text.strip()) < 10:
        raise FileReadError(
            "PDFからテキストを抽出できませんでした。"
            "スキャンPDF（画像PDF）は現在非対応です。"
        )


    # テキスト量バリデーション
    validate_text_size(full_text, "pdf")


    return full_text
```

```python
"""
app/core/excel_writer.py に追加


テンプレートのアップロード時に危険な要素を検出して警告する。
"""


import openpyxl
import io
from app.core.safe_logger import get_logger


logger = get_logger(__name__)




def validate_template(template_bytes: bytes) -> dict:
    """
    テンプレートの内容を検証し、警告事項を返す。
    
    Returns:
        {
            "valid": True/False,
            "warnings": ["警告メッセージ1", "警告メッセージ2"],
            "errors": ["エラーメッセージ1"],
            "sheet_count": 3,
            "has_formulas": True,
            "has_merged_cells": True,
        }
    """
    result = {
        "valid": True,
        "warnings": [],
        "errors": [],
        "sheet_count": 0,
        "has_formulas": False,
        "has_merged_cells": False,
    }


    try:
        wb = openpyxl.load_workbook(io.BytesIO(template_bytes))
    except Exception as e:
        result["valid"] = False
        result["errors"].append(f"Excelファイルとして読み込めません: {type(e).__name__}")
        return result


    result["sheet_count"] = len(wb.sheetnames)


    for ws in wb.worksheets:
        # 結合セルの確認
        if ws.merged_cells.ranges:
            result["has_merged_cells"] = True


        # 数式セルの確認
        for row in ws.iter_rows():
            for cell in row:
                if cell.value and isinstance(cell.value, str) and cell.value.startswith("="):
                    result["has_formulas"] = True
                    break


    # ファイルサイズ警告
    if len(template_bytes) > 5 * 1024 * 1024:  # 5MB超
        result["warnings"].append(
            "テンプレートファイルが5MBを超えています。"
            "処理に時間がかかる可能性があります。"
        )


    # シート数警告
    if result["sheet_count"] > 10:
        result["warnings"].append(
            f"シート数が{result['sheet_count']}枚あります。"
            "処理対象のシートのみを含むファイルに整理することを推奨します。"
        )


    wb.close()
    return result
```

## 3.3 [警告] LLM出力のJSON抽出強化


```python
"""
app/core/extractor.py の _parse_llm_response を強化
"""


import re
import json
from typing import Optional




def extract_json_from_response(raw_response: str) -> dict:
    """
    LLMのレスポンスからJSON部分を強制的に抽出する。
    
    対応パターン:
      1. 純粋なJSON
      2. ```json ... ``` で囲まれたJSON
      3. ```  ... ``` で囲まれたJSON
      4. 「以下が結果です：」等のテキスト + JSON
      5. JSON + 「以上です」等の後続テキスト
      6. 複数のJSONブロックがある場合は最初のものを使用
    """


    text = raw_response.strip()


    # パターン1: 純粋なJSON
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        pass


    # パターン2-3: コードブロックで囲まれたJSON
    code_block_match = re.search(
        r'```(?:json)?\s*\n?(.*?)\n?\s*```',
        text,
        re.DOTALL,
    )
    if code_block_match:
        try:
            return json.loads(code_block_match.group(1).strip())
        except json.JSONDecodeError:
            pass


    # パターン4-5: テキスト中の最初の { ... } を抽出
    brace_depth = 0
    start_idx = None
    for i, char in enumerate(text):
        if char == '{':
            if brace_depth == 0:
                start_idx = i
            brace_depth += 1
        elif char == '}':
            brace_depth -= 1
            if brace_depth == 0 and start_idx is not None:
                json_candidate = text[start_idx:i + 1]
                try:
                    return json.loads(json_candidate)
                except json.JSONDecodeError:
                    start_idx = None
                    continue


    # パターン6: 最初の [ ... ] を抽出（配列形式の場合）
    bracket_depth = 0
    start_idx = None
    for i, char in enumerate(text):
        if char == '[':
            if bracket_depth == 0:
                start_idx = i
            bracket_depth += 1
        elif char == ']':
            bracket_depth -= 1
            if bracket_depth == 0 and start_idx is not None:
                json_candidate = text[start_idx:i + 1]
                try:
                    parsed = json.loads(json_candidate)
                    if isinstance(parsed, list) and len(parsed) > 0:
                        # 配列の場合は最初の要素がdictならそれを返す
                        if isinstance(parsed[0], dict):
                            return parsed[0]
                        return {"data": parsed}
                except json.JSONDecodeError:
                    start_idx = None
                    continue


    # すべて失敗
    raise ValueError(
        f"レスポンスからJSONを抽出できませんでした。"
        f"レスポンス先頭100文字: {text[:100]}"
    )
```

## 4.1 [致命的] 相関ID（Correlation ID）によるトレーサビリティ

```python
"""
app/api/middleware/correlation_id.py


各リクエストに一意のIDを付与し、
全ログ・レスポンスに含める。
データは保存せずとも、「何が起きたか」を追跡可能にする。
"""


import uuid
import contextvars
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request


# コンテキスト変数（リクエストごとに異なる値を保持）
correlation_id_var: contextvars.ContextVar[str] = contextvars.ContextVar(
    "correlation_id", default=""
)




class CorrelationIdMiddleware(BaseHTTPMiddleware):
    """
    リクエストごとに一意のCorrelation IDを生成し、
    コンテキスト変数に保存する。
    レスポンスヘッダーにも含める。
    """


    async def dispatch(self, request: Request, call_next):
        # リクエストヘッダーにあればそれを使用（分散トレーシング対応）
        cid = request.headers.get("X-Correlation-ID", str(uuid.uuid4())[:12])


        # コンテキスト変数にセット
        token = correlation_id_var.set(cid)


        try:
            response = await call_next(request)
            # レスポンスヘッダーに付与
            response.headers["X-Correlation-ID"] = cid
            return response
        finally:
            correlation_id_var.reset(token)




def get_correlation_id() -> str:
    """現在のリクエストのCorrelation IDを取得する"""
    return correlation_id_var.get("")
```

```python
            output_tokens=extraction_result.completion_tokens,
            extracted_field_count=len(extraction_result.data),
            extraction_time_seconds=round(time.monotonic() - step_start, 3),
            # ❌ 抽出されたデータの値は記録しない
            # ✅ フィールド数だけ記録する
        )


        # コスト追跡
        cost_tracker.record_usage(
            extraction_result.prompt_tokens or 0,
            extraction_result.completion_tokens or 0,
        )


        # Step 8: マスキング復元
        unmasked_data = _unmask_extracted_data(
            extraction_result.data,
            masking_result.mask_map,
        )


        # Step 9: 後処理（正規化）
        normalized_data = _normalize_extracted_data(unmasked_data)


        # Step 10: Excel書き込み
        step_start = time.monotonic()
        output_buffer = write_to_template(
            template_bytes,
            normalized_data,
            parsed_mapping,
        )


        logger.info(
            "excel_write_complete",
            output_size_bytes=output_buffer.getbuffer().nbytes,
            write_time_seconds=round(time.monotonic() - step_start, 3),
        )


        # Step 11: レシート生成
        total_time = time.monotonic() - start_time
        receipt = generate_receipt(
            source_hash=hashlib.sha256(source_bytes).hexdigest(),
            template_hash=hashlib.sha256(template_bytes).hexdigest(),
            source_type=file_type.value,
            source_size=len(source_bytes),
            processing_time=total_time,
        )


        logger.info(
            "processing_complete",
            total_time_seconds=round(total_time, 3),
            receipt_id=receipt.receipt_id,
        )


        # Step 12: レスポンス返却
        output_buffer.seek(0)
        return StreamingResponse(
            output_buffer,
            media_type="application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
            headers={
                "Content-Disposition": f'attachment; filename="excelforge_output.xlsx"',
                "X-ExcelForge-Receipt": receipt.model_dump_json(),
                "X-Correlation-ID": cid,
            },
        )


    except AppException as e:
        logger.error(
            "processing_failed_expected",
            error_code=e.error_code.value,
            error_message=e.message[:100],
            # ❌ リクエストの中身は記録しない
        )
        raise


    except Exception as e:
        logger.error(
            "processing_failed_unexpected",
            error_type=type(e).__name__,
            error_message=str(e)[:100],
        )
        raise


    finally:
        # Step 13: クリーンアップ
        cleanup_variables(
            source_bytes, template_bytes, source_text,
            masking_result, extracted_data, unmasked_data,
            normalized_data, output_buffer, receipt,
        )
        gc.collect()


        logger.info("cleanup_complete")
```

## 4.2 [警告] 顧客テンプレート・設定の管理アーキテクチャ


### 問題の本質


```
現状の設計では、顧客が増えるたびに：
  1. mapping_config JSONを手動作成
  2. コードベースに追加
  3. 再デプロイ


これでは10社になった時点で運用が破綻する。
かつ、データベースは使わない制約がある。
```


### 対応：GCSからのコンフィグ動的ロード


```
設計変更:
  「データベースは使わない」「ストレージは使わない」の制約のうち、
  顧客設定ファイル（JSONのみ）の保管に限りGCSを使用する。


  ただし以下を厳守する：
  ・GCSに保管するのは設定ファイル（JSON）のみ
  ・顧客のアップロードデータは一切保管しない
  ・テンプレートExcelもGCSに保管する（オプション）


  「データを持たない」の定義を以下に修正する：
  ・「顧客の業務データ（PDF/CSV/Excel）を保持しない」
  ・「設定ファイル・テンプレートは保管する」
```


### アーキテクチャ


```
GCS バケット: excelforge-configs/
├── customers/
│   ├── customer_a/
│   │   ├── config.json          ← マッピング設定
│   │   └── template.xlsx        ← テンプレート（任意）
│   ├── customer_b/
│   │   ├── config.json
│   │   └── template.xlsx
│   └── customer_c/
│       └── config.json          ← テンプレートなし（ユーザーがアップロード）
└── defaults/
    └── default_config.json      ← デフォルト設定
```


### 顧客設定ローダー


```python
"""
app/core/config_loader.py


顧客ごとの設定をGCSから動的に読み込む。
起動時にキャッシュし、定期的にリフレッシュする。
"""


import json
import io
import time
import threading
from typing import Optional
from google.cloud import storage
from app.core.safe_logger import get_logger
from app.config import settings


logger = get_logger(__name__)




class CustomerConfig:
    """顧客ごとの設定"""


    def __init__(self, data: dict):
        self.customer_id: str = data.get("customer_id", "")
        self.customer_name: str = data.get("customer_name", "")
        self.mapping_config: Optional[dict] = data.get("mapping_config")
        self.extraction_prompt_override: Optional[str] = data.get("extraction_prompt_override")
        self.post_processing: dict = data.get("post_processing", {})
        self.template_gcs_path: Optional[str] = data.get("template_gcs_path")




class ConfigLoader:
    """
    GCSから顧客設定を読み込み、メモリにキャッシュする。


    設計:
      ・起動時に全顧客の設定を読み込む
      ・10分ごとにバックグラウンドリフレッシュ
      ・GCSアクセス失敗時は既存キャッシュを使い続ける
      ・顧客のアップロードデータは一切GCSに保存しない
    """


    def __init__(
        self,
        bucket_name: str,
        refresh_interval_seconds: int = 600,
    ):
        self._bucket_name = bucket_name
        self._refresh_interval = refresh_interval_seconds
        self._configs: dict[str, CustomerConfig] = {}
        self._template_cache: dict[str, bytes] = {}
        self._lock = threading.Lock()
        self._client: Optional[storage.Client] = None


        # 初回読み込み
        self._refresh()


        # バックグラウンドリフレッシュ
        self._thread = threading.Thread(
            target=self._background_refresh,
            daemon=True,
        )
        self._thread.start()


    def _get_client(self) -> storage.Client:
        if self._client is None:
            self._client = storage.Client()
        return self._client


    def _refresh(self) -> None:
        """GCSから全顧客設定を再読み込みする"""
        try:
            client = self._get_client()
            bucket = client.bucket(self._bucket_name)


            new_configs = {}
            blobs = bucket.list_blobs(prefix="customers/")


            for blob in blobs:
                if blob.name.endswith("/config.json"):
                    try:
                        content = blob.download_as_text()
                        data = json.loads(content)
                        customer_id = data.get("customer_id", "")
                        if customer_id:
                            new_configs[customer_id] = CustomerConfig(data)
                    except Exception as e:
                        logger.warning(
                            "config_load_failed",
                            blob_name=blob.name,
                            error=str(e)[:100],
                        )


            with self._lock:
                self._configs = new_configs


            logger.info(
                "configs_refreshed",
                customer_count=len(new_configs),
            )


        except Exception as e:
            logger.error(
                "config_refresh_failed",
                error_type=type(e).__name__,
                error_message=str(e)[:100],
            )


    def _background_refresh(self) -> None:
        while True:
            time.sleep(self._refresh_interval)
            self._refresh()


    def get_config(self, customer_id: str) -> Optional[CustomerConfig]:
        """顧客IDから設定を取得する"""
        with self._lock:
            return self._configs.get(customer_id)


    def get_template(self, customer_id: str) -> Optional[bytes]:
        """
        顧客のテンプレートExcelをGCSから取得する。
        メモリにキャッシュする（再起動で消える）。
        """
        # キャッシュチェック
        if customer_id in self._template_cache:
            return self._template_cache[customer_id]


        config = self.get_config(customer_id)
        if config is None or config.template_gcs_path is None:
            return None


        try:
            client = self._get_client()
            bucket = client.bucket(self._bucket_name)
            blob = bucket.blob(config.template_gcs_path)
            template_bytes = blob.download_as_bytes()


            # キャッシュに保存
            with self._lock:
                self._template_cache[customer_id] = template_bytes


            return template_bytes


        except Exception as e:
            logger.error(
                "template_download_failed",
                customer_id=customer_id,
                error=str(e)[:100],
            )
            return None


    @property
    def customer_count(self) -> int:
        with self._lock:
            return len(self._configs)




# シングルトン
config_loader = ConfigLoader(
    bucket_name=f"{settings.GCP_PROJECT_ID}-excelforge-configs",
    refresh_interval_seconds=600,
)
```

## 一言まとめ


> **「副業でも本番品質」を実現するために追加したのは、5つのミドルウェアと3つの管理モジュール。月額コストの増加はGCSの数円分だけ。しかしこれにより、「ログからの情報漏洩」「OOMクラッシュ」「クラウド破産」「障害調査不能」「顧客追加のたびに再デプロイ」——すべての致命的リスクが構造的に排除された。セキュリティと運用性は、後から足すものではなく、最初から設計に組み込むもの。**
